Scope of var, let, and const:

var:
1. var declarations are function-scoped, which means they are only accessible within the function where they are defined.
2. If you declare a variable with var outside of a function, it becomes a global variable and is accessible throughout the entire program.
3. var variables are hoisted, meaning the declaration is moved to the top of the function or global scope during compilation, but the assignment remains in place. This can lead to unexpected behavior.

let:
1. let declarations are block-scoped, which means they are only accessible within the nearest enclosing pair of curly braces {} (e.g., if statements, loops, functions, etc.).
2. Unlike var, let variables are not hoisted, so you can't access them before their declaration in the code.

const:
1. const also has block scope like let, and it is used for variables whose values should not be reassigned after initialization.
2. Like let, const variables are not hoisted, and they must be initialized at the time of declaration.

Use Cases of null and undefined:

undefined:
1. undefined is a primitive value in JavaScript that represents the absence of a value or an uninitialized variable.
2. Variables that have been declared but not assigned a value are automatically initialized to undefined.
3. It can also be the default return value of a function that does not explicitly return anything.

null:
1. null is a special value in JavaScript that represents the intentional absence of any object value.
2. It is often used when you want to explicitly indicate that a variable or property should have no value.
3. It is commonly used in situations where you want to reset an object or variable to an empty or non-existent state.

REST API:

REST stands for Representational State Transfer, and a REST API (Application Programming Interface) is an architectural style for designing networked applications.
